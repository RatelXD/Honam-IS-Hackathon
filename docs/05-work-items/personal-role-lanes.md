# 개인 사전 탐색 역할 레인

이 문서는 팀원 배정표가 아니라, 개인이 해커톤 시작 전에 feasibility와 MVP 범위를 검증하기 위한 역할 레인 구조다. 실제 팀 프로젝트가 시작되기 전까지 사람 이름을 요구하지 않으며, 이름 기반 소유권 대신 레인 기반 이슈와 산출물로 진행 상태를 관리한다.

## 원칙

- 현재 단계는 개인 사전 탐색이다. 팀원별 담당 업무를 확정하지 않는다.
- 레인은 탐색 관점을 나누기 위한 작업 분류이며, 실제 팀 구성 이후에만 실명 또는 GitHub 계정을 매핑한다.
- MVP는 광주 공식 출처 기반 RAG 웹 데모로 둔다.
- 대상 사용자는 초기 정착 외국인이며, 한국어 공식 원문, 쉬운 한국어, 영어, 베트남어, 중국어 흐름을 고려한다.
- 개인정보와 안전을 기본값으로 둔다. 민감정보를 수집하지 않고, 저장이 필요한 경우 최소화와 삭제 경로를 먼저 정의한다.

## 역할 레인

### Frontend

목표: 외국인 사용자가 질문하고 답변 근거를 확인할 수 있는 웹 데모 흐름을 탐색한다.

초기 산출물:

- 질문 입력, 언어 선택, 답변, 공식 출처 링크, 면책/안전 문구가 포함된 화면 구조
- 모바일 우선 접근성 점검 항목
- 쉬운 한국어와 다국어 표시 방식
- 실패 시 사용자에게 보여줄 안전한 대체 메시지

### Backend

목표: RAG 호출, API 라우팅, 입력 검증, 에러 처리, 제한 정책의 최소 백엔드 구조를 탐색한다.

초기 산출물:

- API 엔드포인트 후보와 요청/응답 스키마 초안
- LLM 호출 타임아웃, 재시도, rate limit, CORS 정책
- 모델 실패 또는 출처 부족 시 안전 응답 규칙
- 배포 환경변수와 secret 주입 방식 체크

### Data/RAG

목표: 광주 공식 출처만으로 MVP 답변 근거를 만들 수 있는지 검증한다.

초기 산출물:

- 공식 출처 인벤토리와 우선순위
- 수집 대상 문서의 언어, 갱신일, 사용 조건, URL 목록
- chunking, 임베딩, 검색 품질 평가 기준
- 한국어 원문에서 쉬운 한국어/영어/베트남어/중국어로 설명할 때의 용어집 후보

### PM/Docs

목표: 개인 탐색을 실행 가능한 이슈와 마일스톤으로 정리한다.

초기 산출물:

- MVP 범위, 비범위, 리스크, 데모 시나리오
- 첫 실행 이슈의 모델 우선순위 결정 절차
- pre-execution gate 체크리스트
- 심사위원에게 설명 가능한 문제 정의와 가치 제안

### QA/Demo

목표: 데모가 개인정보, 언어, 출처, 배포 실패 조건에서 안전하게 동작하는지 검증한다.

초기 산출물:

- 공식 출처가 있는 질문/없는 질문 테스트 케이스
- 다국어 답변 품질과 용어 일관성 확인
- 개인정보 입력 유도 방지 확인
- 데모 증거: 화면 캡처, 재현 가능한 입력, 출처 링크, 실패 처리 기록

## 마일스톤 구조

### M0: Pre-execution gates

팀 구현 전에 닫아야 하는 준비 마일스톤이다.

필수 이슈:

1. 공식 출처 인벤토리 확정
2. OpenAI/Gemini 모델 우선순위 결정
3. 배포 계정과 권한 확인
4. 다국어 용어집 초안 작성
5. 개인정보/안전 기본 정책 확정

### M1: RAG MVP skeleton

공식 출처 기반으로 질문-검색-답변-근거 표시가 이어지는 최소 흐름을 만든다.

필수 이슈:

- Frontend: 질문 입력과 출처 표시 UI
- Backend: 질의 API와 LLM 실패 처리
- Data/RAG: 공식 문서 chunking과 검색 품질 확인
- QA/Demo: 대표 질문 세트와 데모 evidence 수집

### M2: Multilingual demo hardening

한국어 원문을 기반으로 쉬운 한국어, 영어, 베트남어, 중국어 사용자 흐름을 검증한다.

필수 이슈:

- 언어별 답변 톤과 용어집 적용
- 출처 부족/모델 실패/과도한 개인정보 입력 상황 처리
- 배포 환경에서 CORS, timeout, rate limit 확인

## 이슈 구조

각 이슈는 사람 이름 대신 레인을 지정한다.

권장 필드:

- Role lane: Frontend / Backend / Data/RAG / PM/Docs / QA/Demo
- Milestone: M0 / M1 / M2
- Problem: 해결하려는 사용자 또는 운영 문제
- Scope: 이번 이슈에서 끝낼 범위
- Acceptance criteria: 완료 판단 기준
- Demo evidence: 화면, 로그, 링크, 입력/출력 예시 등 재현 증거
- Privacy/data checks: 개인정보 수집 여부, 저장 여부, 삭제/비저장 정책
- Language checks: 한국어 원문, 쉬운 한국어, 영어, 베트남어, 중국어 영향
- Deploy checks: env var, secret, CORS, timeout, rate limit, failure path

## 실명 매핑 규칙

- 개인 사전 탐색 단계에서는 실명, 학번, 연락처, 팀원 이름을 이슈 필수값으로 두지 않는다.
- 팀 프로젝트가 시작되고 역할 동의가 끝난 뒤에만 레인을 사람 또는 GitHub 계정에 매핑한다.
- 매핑 이후에도 이슈 본문은 레인과 산출물을 중심으로 유지한다.
